# mittwald Agent Skills

mittwald stellt spezialisierte Agent Skills bereit, die KI-Assistenten durch komplexe Workflows wie Migrationen und Deployments auf mStudio führen. Die Skills liefern strukturierte Schritt-für-Schritt-Anleitungen und Fachwissen ohne aufwendige individuelle Prompts.

## Was sind Agent Skills?

Agent Skills sind vorgefertigte Ablaufvorlagen, mit denen KI-Assistenten komplexe, mehrstufige Prozesse umsetzen können. Statt jedes Mal alles neu zu erklären, startest du einen Skill und wirst konversationell durch den gesamten Ablauf geführt.

**Klassischer Ansatz:**
> "Projekt erstellen, Node.js konfigurieren, MySQL einrichten, Backup-Plan anlegen..."

**Mit Agent Skills:**
> "Deploye meine Express-App auf mittwald"

Der Skill nimmt dir die Komplexität des Ablaufs ab, stellt bei Bedarf Rückfragen und führt den kompletten Prozess aus.

## Verfügbare Skills

### mittwald-migrate

Leitet dich durch eine konversationelle, phasenweise Migration beliebiger Workloads zu mittwald mStudio.

**Quellen:** Kubernetes, Docker Compose, VPS, andere Hoster oder bestehende mStudio-Projekte

**Workflow:** Discovery → Plan → Provision → Migrate → Verify → Cutover

**Trigger:** "migrate to mittwald", "Umzug nach mittwald", "move to mStudio"

[Mehr erfahren →](./mittwald-migrate)

### mittwald-zerodeploy

Zero-Config-Deployment zu mStudio Container Hosting mit Railpack Auto-Erkennung. Kein Dockerfile nötig.

**Workflow:** Provision → Local CLI Deploy → GitHub Actions → Verify

**Trigger:** "deploy to mittwald", "help me deploy", "set up mittwald deployment"

[Mehr erfahren →](./mittwald-zerodeploy)

## Installation

### Voraussetzungen

Vor der Nutzung der Skills:

- Aktiver mittwald mStudio-Account
- API-Token mit Rolle `api_write` aus den [mStudio Profileinstellungen](https://studio.mittwald.de/profile/api-token)
- Installierte mittwald CLI ([Installationsanleitung](/docs/v2/cli/))

### Installationsanleitung

#### Für VS Code Copilot

**Persönliche Skills (empfohlen):**

```shellsession
user@local $ mkdir -p ~/.agents/skills
user@local $ git clone https://github.com/mittwald/agent-skills.git ~/.agents/skills/mittwald-skills
```

**Projektlokale Skills:**

```shellsession
user@local $ mkdir -p .agents/skills
user@local $ git clone https://github.com/mittwald/agent-skills.git .agents/skills/mittwald-skills
```

#### Für Claude Code

```shellsession
user@local $ mkdir -p ~/.claude/skills
user@local $ git clone https://github.com/mittwald/agent-skills.git ~/.claude/skills/mittwald-skills
```

Oder über den Plugin Marketplace:

```shellsession
user@local $ /plugin marketplace add mittwald/agent-skills
user@local $ /plugin install mittwald-migrate@agent-skills
```

#### Für andere KI-Assistenten

Prüfe die Skill-Installationspfade in der Dokumentation deines Assistenten und clone das Repository dorthin.

## Nutzung

Nach der Installation triggerst du die Skills direkt im Chat mit deinem KI-Assistenten:

**Für Migrationen:**
> "Ich möchte meine WordPress-Seite von Hetzner zu mittwald migrieren."

**Für Deployments:**
> "Hilf mir, meine Express-App auf mittwald zu deployen"

Der Assistent lädt den passenden Skill und führt dich mit Kontextfragen durch den Workflow.

## Authentifizierung

Die Skills brauchen mindestens eine Methode, um mit mStudio zu sprechen. Du musst nicht alle drei einrichten:

### Option 1: MCP Server (Empfohlen)

Konfiguriere den [mittwald MCP-Server](/docs/v2/agentic-integration/mcp/) in deinem KI-Assistenten. Die Skills rufen dann `mcp__mittwald__mittwald_*` Tools direkt auf.

**Am besten für:** KI-Assistenten mit MCP-Support

### Option 2: mittwald CLI

Authentifiziere die CLI lokal:

```shellsession
user@local $ mw login token
user@local $ mw user get
```

**Am besten für:** Terminal-basierte Workflows

### Option 3: HTTP API

Setze lokal eine Umgebungsvariable:

```shellsession
user@local $ export MITTWALD_API_TOKEN=<dein-api_write-token>
```

**Am besten für:** Skripte und Automatisierung

[Mehr zur Authentifizierung →](./authentication)
