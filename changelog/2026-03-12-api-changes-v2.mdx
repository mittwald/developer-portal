## Summary


- The response property type for `blacklist/items/` and `whitelist/items/` in the `GET /v2/projects/{projectId}/mail-settings` operation has changed from `string`/`idn-email` to `string`/`''` for status `200`. This is a breaking change.
- The request body for the `PATCH /v2/projects/{projectId}/mail-settings/{mailSetting}` and `PUT /v2/projects/{projectId}/mail-settings/{mailSetting}` operations has had `subschema #1` and `subschema #2` removed from the `oneOf` list. This is a breaking change.
- The request property type for `blacklist/items/` and `whitelist/items/` in the `PUT /v2/projects/{projectId}/mailsettings/blacklist` and `PUT /v2/projects/{projectId}/mailsettings/whitelist` operations has changed from `string`/`idn-email` to `string`/`''`. This is a breaking change.
- A new pattern `^([a-zA-Z0-9.!#$%&'*\\/=?^_{|}~-]+@[a-zA-Z0-9-*]+(?:\\.([a-zA-Z0-9-]+|\\*))+)$` has been added to the request properties `blacklist/items/` and `whitelist/items/` in the `PUT /v2/projects/{projectId}/mailsettings/blacklist` and `PUT /v2/projects/{projectId}/mailsettings/whitelist` operations.

- A new optional request parameter `types` has been added to the `GET /v2/orders` operation.

## Client package releases


### mittwald PHP SDK Release v2.1.196

The latest release of the mittwald PHP SDK, version 2.1.196, introduces several new features and enhancements aimed at improving user experience and functionality. 

#### New Features

- **Backup Client**
  - A new method to update a backup's expiration time has been added, allowing users to manage backup lifecycles more effectively.
  
- **Container Client**
  - Users can now list stacks specific to their account, enhancing stack management and organization.

#### Enhancements

- **Backup Client**
  - Error handling for the new backup expiration method has been improved, with detailed responses for various error scenarios.

- **Container Client**
  - Enhanced error responses for listing self stacks, covering a range of client errors.

- **Key Class in AI Hosting**
  - The `models` field has been deprecated, with documentation updates guiding users to the correct route for model information.

#### Bug Fixes

- Various minor issues have been addressed, and validation schemas have been improved to ensure better compliance and smoother API interactions.

For more details, visit the [release page](https://github.com/mittwald/api-client-php/releases/tag/v2.1.196).

### mittwald PHP SDK Release v2.1.195

The latest release of the mittwald PHP SDK, version v2.1.195, introduces several updates and improvements. 

#### Features
- A new schema for the verify address response has been introduced, enhancing the response structure and refining request body handling.

#### Changes
- **VerificationVerifyAddressOKResponse**:
  - The response body now uses `VerificationVerifyAddressOKResponseBody` instead of `VerifyAddressResponse`.
  - The `body` property explicitly defines its structure, requiring an `exists` field of type boolean.
  - The cloning mechanism for the `body` property has been updated to ensure deep copies are maintained.

- **VerificationVerifyAddressOKResponseBody**:
  - Renamed from `VerifyAddressResponse`.
  - Methods have been adjusted to align with the new response format and naming conventions.

- **VerificationVerifyAddressRequest**:
  - The `country` field description has been updated to specify the use of the ISO 3166-2 Alpha 2 country code format.

- **VerificationVerifyAddressRequestBody**:
  - The `country` field description has been consistently updated to reflect the ISO 3166-2 Alpha 2 country code requirement.

#### Removed
- The `VerifyAddressConfidence` enum has been removed, simplifying the verification process.

For more details, visit the [release page](https://github.com/mittwald/api-client-php/releases/tag/v2.1.195).

### mittwald PHP SDK Release v2.1.194

The latest release of the mittwald PHP SDK, version v2.1.194, introduces several updates and enhancements. 

#### Added
- A `refreshToken` field has been introduced in the `ChangePasswordOKResponse` and `ChangePasswordOKResponseBody` classes. This token allows users to refresh their access token even after it has expired.

#### Updated
- The required fields for responses now include the new `refreshToken`.
- The constructor for `ChangePasswordOKResponseBody` now accepts a `refreshToken` as an argument.
- A `getRefreshToken` method has been added to retrieve the `refreshToken`.
- A `withRefreshToken` method has been included to facilitate fluent interface usage while updating the `refreshToken`.
- The hydration method has been updated to correctly process the new `refreshToken` from input data.
- The output serialization method now includes the `refreshToken` in the response output.

For more details, visit the [release page](https://github.com/mittwald/api-client-php/releases/tag/v2.1.194).
