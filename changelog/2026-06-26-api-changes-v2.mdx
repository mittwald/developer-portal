This week, the mittwald API introduces several significant changes, including the removal of multiple required and optional properties from the responses of various endpoints, updates to response body formats, the addition of new endpoints for managing container templates and customer projects, and the introduction of new optional request parameters.

:::caution Breaking changes

This document contains changes that can under certain circumstances be considered breaking. Please review the changes carefully.

While we generally strive to avoid breaking changes in accordance with our [API stability guarantees](../../../../../docs/v2/api/stability), we occasionally might perform changes that would be considered breaking in the strictest sense of the term, but we do not consider as breaking in a practical sense. We will always provide ample notice and documentation for such changes.

:::



## Summary


- The property type/format of the response for the `GET /v2/conversations` operation has changed from 'object' to '' for status '200'. This is a breaking change.
- The following required properties have been removed from the response with status '200' for the `GET /v2/conversations` operation: `/items/conversationId`, `/items/createdAt`, `/items/mainUser`, `/items/shortId`, `/items/status`, `/items/title`, and `/items/visibility`. This is a breaking change.
- The response body type/format for the `GET /v2/conversations/{conversationId}` operation has changed from 'object' to '' for status '200'. This is a breaking change.
- The following required properties have been removed from the response with status '200' for the `GET /v2/conversations/{conversationId}` operation: `conversationId`, `createdAt`, `mainUser`, `shortId`, `status`, `title`, and `visibility`. This is a breaking change.
- The following optional properties have been removed from the response with status '200' for the `GET /v2/conversations` operation: `/items/category`, `/items/createdBy`, `/items/lastMessage`, `/items/lastMessageAt`, `/items/lastMessageBy`, `/items/notificationRoles`, `/items/relatedTo`, `/items/relations`, and `/items/sharedWith`.
- The following optional properties have been removed from the response with status '200' for the `GET /v2/conversations/{conversationId}` operation: `category`, `createdBy`, `lastMessage`, `lastMessageAt`, `lastMessageBy`, `notificationRoles`, `relatedTo`, `relations`, and `sharedWith`.
- The following optional properties have been removed from the response with status '200' for the `GET /v2/conversations/{conversationId}/members` operation: `/items/allOf[#/components/schemas/de.mittwald.v1.conversation.User]/atlasGroup`, `/items/allOf[#/components/schemas/de.mittwald.v1.conversation.User]/department`, and `/items/allOf[#/components/schemas/de.mittwald.v1.conversation.User]/group`.
- The following optional properties have been removed from the response with status '200' for the `GET /v2/conversations/{conversationId}/messages` operation: `/items/oneOf[#/components/schemas/de.mittwald.v1.conversation.Message]/createdBy/atlasGroup`, `/items/oneOf[#/components/schemas/de.mittwald.v1.conversation.Message]/createdBy/department`, `/items/oneOf[#/components/schemas/de.mittwald.v1.conversation.Message]/createdBy/group`, and several properties related to status updates.
- The schemas `de.mittwald.v1.conversation.Department` and `de.mittwald.v1.conversation.Group` have been removed from the API.
- New endpoints have been added: 
  - `GET /v2/container-templates` to list Container Templates.
  - `GET /v2/container-templates/{templateId}` to get a Container Template by ID.
  - `GET /v2/customers/{customerId}/projects` to list Projects belonging to a Customer.
  - `GET /v2/projects/{projectId}/domain-migrations` to list Domain-Migrations belonging to a Project.
  - `POST /v2/projects/{projectId}/stacks` to create a Stack.
  - `DELETE /v2/stacks/{stackId}` to delete a Stack.
- A new optional request parameter `state` has been added to the `GET /v2/contributors/{contributorId}/webhook-executions` operation.
- The following optional properties have been added to the responses with status '200':
  - `/items/allOf[#/components/schemas/de.mittwald.v1.conversation.BasicConversation, subschema #2]` for the `GET /v2/conversations` operation.
  - `/items/oneOf[#/components/schemas/de.mittwald.v1.conversation.Message]/createdBy/isEmployee` and others for the `GET /v2/conversations/{conversationId}/messages` operation.
- A new optional request property `description` has been added to the `PATCH /v2/stacks/{stackId}` operation.
